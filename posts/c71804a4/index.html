（ai）这个 1111 端口，是你在启动 Docker 容器时，手动将宿主机的 1111 端口映射到容器内部 redroid 的 5555 端口（ADB 服务端口）的结果。
 docker ps -a 的输出0.0.0.0:1111->5555/tcp, :::1111->5555/tcp priceless_knuth

  1. 5555/tcp （容器内部）：这是 redroid 容器内部的 Android 系统监听的 ADB 端口。从你后面执行的 netstat 也能看到，容器内 adbd 确实在监听 5555 端口（[.:]:5555）。
  2. 0.0.0.0:1111 （宿主机外部）：这是你宿主机（就是你截图里登录的 model 这台机器）上的 1111 端口。
  3. -> 的含义：箭头表示一种端口映射或端口转发规则。意思是，任何发送到你宿主机 1111 端口的网络请求，都会被自动转发到这个 priceless_knuth 容器内部的 5555 端口。

ip a 和 ifconfig 区别

对比维度 ip a (或 ip addr) ifconfig
所属工具包 iproute2  net-tools 
内核通信 Netlink sockets (更高效的新机制)  ioctl (传统的系统调用) 
信息显示 默认显示所有接口（包括已禁用的），并明确标注接口状态（UP/DOWN）。 默认只显示已启用的接口。要显示所有接口（包括禁用的），需加 -a 参数。
IPv6支持 原生支持，IPv6地址会清晰地显示在 inet6 行。 支持有限，显示较为繁琐。
主要优势 功能统一且强大，是网络配置的“一站式”工具。输出信息更结构化，利于脚本处理。 命令简单直观，输出格式对新手友好，易于阅读。
主要劣势 语法稍显复杂，需要记住子命令（如 addr, link）。 功能较弱，已过时（缺乏对网络命名空间等新特性的支持），且在不同的Linux发行版上行为可能有细微差异。
未来趋势 主流和推荐，几乎所有现代Linux发行版都预装并推荐使用。 逐渐被淘汰，部分最小化安装的系统中可能不存在。
